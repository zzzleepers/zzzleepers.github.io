背包问题详解（含优化推演步骤）

一、0/1 背包问题

1. 问题定义

  • 输入：n 件物品（体积v[i]、价值w[i]），背包容量 M。

  • 约束：每件物品只能选 0 或 1 次。

  • 目标：最大化背包内物品总价值。

2. 动态规划原理

状态定义

dp[i][m]表示前 i 件物品在背包容量为 m时的最大价值。

状态转移推演

  • 考虑第 i 件物品时，有两种选择：

  1. 不选第 i 件：价值与前 i-1 件物品在容量 m 时相同，即dp[i][m] = dp[i-1][m]。

  2. 选第 i 件：需从容量 m 中扣除v[i]，价值为前 i-1 件在容量m-v[i]时的价值加上w[i]，即dp[i][m] = dp[i-1][m-v[i]] + w[i]。

  • 取两种情况的最大值：dp[i][m] = max(dp[i-1][m], dp[i-1][m-v[i]] + w[i])。

3. 空间优化推演（滚动数组）

原始二维数组的问题

  • 空间复杂度O(n*M)，当 n 和 M 较大时（如 1e4），内存占用过高。

优化思路

  • 观察转移方程：dp[i][m]仅依赖dp[i-1][...]（上一行数据），与更早的行无关。

  • 可复用数组空间：用一维数组dp[M]，每次计算第 i 件物品时覆盖上一行数据。

关键：遍历顺序调整

  • 若正序遍历 m（从小到大）：计算dp[m]时，dp[m-v[i]]已被当前物品更新（相当于重复选了第 i 件），违反 0/1 约束。

  • 必须倒序遍历 m（从大到小）：确保dp[m-v[i]]是上一行的旧值（未被当前物品更新）。

优化后转移方程

dp[m] = max(dp[m], dp[m-v[i]] + w[i])（m从M到v[i]）。

4. 核心代码

// 二维数组版本int dp[1005][1005];for (int i = 1; i <= n; i++) {    for (int m = 1; m <= M; m++) {        dp[i][m] = dp[i-1][m]; // 不选第i件        if (m >= v[i]) {            dp[i][m] = max(dp[i][m], dp[i-1][m-v[i]] + w[i]); // 选第i件        }    }}// 一维优化版本int dp[1005];for (int i = 1; i <= n; i++) {    for (int m = M; m >= v[i]; m--) { // 倒序遍历        dp[m] = max(dp[m], dp[m-v[i]] + w[i]);    }}

二、完全背包问题

1. 问题定义

  • 差异：每件物品可无限次选择（数量不限）。

2. 动态规划原理

状态转移推演

  • 与 0/1 背包的核心区别：选第 i 件物品后，仍可继续选该物品。

  • 转移方程调整：

  • 选第 i 件物品时，依赖当前 i 的子状态（而非 i-1），即dp[i][m] = dp[i][m-v[i]] + w[i]（允许重复选择）。

  • 完整方程：dp[i][m] = max(dp[i-1][m], dp[i][m-v[i]] + w[i])。

3. 空间优化推演

一维数组适配

  • 同样可优化为一维数组，但遍历顺序需调整：

  • 若正序遍历 m（从小到大）：dp[m-v[i]]已包含当前物品的选择（可能已选过 1 次或多次），符合 “无限选取” 逻辑。

  • 举例：m=5、v[i]=2时，dp[5]依赖dp[3]，而dp[3]可能已选过 1 次该物品，因此dp[5]相当于 “再选 1 次”。

优化后转移方程

dp[m] = max(dp[m], dp[m-v[i]] + w[i])（m从v[i]到M）。

4. 核心代码

int dp[1005];for (int i = 1; i <= n; i++) {    for (int m = v[i]; m <= M; m++) { // 正序遍历        dp[m] = max(dp[m], dp[m-v[i]] + w[i]);    }}

三、多重背包问题

1. 问题定义

  • 约束：每件物品最多选k[i]次。

2. 解法一：暴力法

原理推演

  • 将 “最多 k 个物品” 拆分为 “k 个独立的 0/1 物品”，例如k=3拆分为 3 个相同物品。

  • 直接套用 0/1 背包解法，时间复杂度O(nMk)。

核心代码

int dp[1005];for (int i = 1; i <= n; i++) {    // 拆分为k[i]个0/1物品    for (int t = 1; t <= k[i]; t++) {        for (int m = M; m >= v[i]; m--) {            dp[m] = max(dp[m], dp[m-v[i]] + w[i]);        }    }}

3. 解法二：二进制优化

原理推演

  • 问题：暴力法中 k 过大时（如 1e5），时间复杂度过高。

  • 优化思路：用 “二进制拆分” 减少物品数量。

  • 任何整数k可表示为2^0 + 2^1 + ... + 2^t + r（r < 2^(t+1)），例如k=10 = 1+2+4+3。

  • 拆分后，log2(k)个 “虚拟物品” 可组合出1~k的所有数量（如1+4=5、2+3=5）。

步骤

  1. 对每件物品，按二进制拆分k[i]为c_1, c_2, ..., c_m（c_j为 2 的幂次或剩余值）。

  2. 每个c_j对应虚拟物品：体积c_jv[i]，价值c_jw[i]。

  3. 用 0/1 背包处理虚拟物品，时间复杂度优化为O(nMlog k)。

核心代码

int dp[1005];for (int i = 1; i <= n; i++) {    int cnt = 1, remain = k[i];    while (remain > 0) {        int take = min(cnt, remain); // 拆分出的数量        // 处理虚拟物品        for (int m = M; m >= take * v[i]; m--) {            dp[m] = max(dp[m], dp[m - take*v[i]] + take*w[i]);        }        remain -= take;        cnt *= 2; // 下一个幂次    }}

4. 解法三：单调队列优化

原理推演

  • 目标：摆脱对k的依赖，时间复杂度优化至O(n*M)。

  • 关键观察：

  • 对物品(v, w, k)，容量m可表示为m = r + t*v（r为余数，0 ≤ r < v；t为倍数）。

  • 转移方程变形：dp[r + tv] = max(dp[r + pv] + (t-p)w)（0 ≤ t-p ≤ k），即max(dp[r + pv] - pw) + tw。

  • 问题转化为：对每个余数r，在滑动窗口[t-k, t]内求dp[r + pv] - pw的最大值（可用单调队列实现）。

步骤

  1. 按余数r分组（0 ≤ r < v），每组包含r, r+v, r+2v, ...。

  2. 对每组，遍历倍数t，用单调队列维护窗口内最大值。

  3. 计算当前容量的最大价值：dp[r + tv] = 队列最大值 + tw。

核心代码

int dp[1005], pre[1005];for (int i = 1; i <= n; i++) {    memcpy(pre, dp, sizeof(dp)); // 保存上一轮结果    int v = v[i], w = w[i], k = k[i];    for (int r = 0; r < v; r++) { // 按余数分组        deque<int> q; // 存储倍数p，维护最大值        for (int t = 0; ; t++) { // 倍数t            int m = r + t*v; // 当前容量            if (m > M) break;            int val = pre[m] - t*w; // 待入队的值            // 维护队列单调递减            while (!q.empty() && (pre[r + q.front()*v] - q.front()*w) <= val) {                q.pop_front();            }            q.push_front(t);            // 移除超出窗口的元素（t-p ≤ k）            while (q.back() < t - k) {                q.pop_back();            }            // 更新dp[m]            int best_p = q.back();            dp[m] = max(dp[m], pre[r + best_p*v] + (t - best_p)*w);        }    }}

四、树形背包问题

1. 问题定义

  • 输入：n 个节点的树（每个节点含 1 个水果），目标 k。

  • 目标：求节点 1 所在连通块恰好有 k 个水果的方案数。

2. 动态规划原理

状态定义

dp[i][j]表示以 i 为根的子树中，i 所在连通块有 j 个水果的方案数。

状态转移推演

  • 合并子树时，若根为u，子树为v：

  • 方案数为 “u 的 j 个水果” 与 “v 的 k-j 个水果” 的乘积之和（卷积）。

  • 方程：dp[u][j] += dp[u][i] * dp[v][j-i]（1 ≤ i < j ≤ k）。

  • 需同时考虑 “切断边” 的情况（即不合并子树，方案数累加原dp[u][j]）。

时间复杂度分析

  • 树上任意两节点仅在其 LCA（最近公共祖先）处合并一次，总操作次数为O(n²)。

3. 核心代码

vector<int> adj[1005];int sz[1005]; // 子树大小long long dp[1005][1005];void dfs(int u, int parent) {    sz[u] = 1;    dp[u][1] = 1; // 初始：仅含自身    for (int v : adj[u]) {        if (v == parent) continue;        dfs(v, u);        // 临时数组存储合并结果        long long temp[1005] = {0};        // 合并子树v到u        for (int i = 1; i <= sz[u]; i++) {            for (int j = 1; j <= sz[v]; j++) {                if (i + j > k) continue;                temp[i + j] += dp[u][i] * dp[v][j];            }        }        // 累加“切断边”的方案        for (int i = 1; i <= sz[u]; i++) {            temp[i] += dp[u][i];        }        // 更新sz和dp        sz[u] += sz[v];        for (int i = 1; i <= sz[u] && i <= k; i++) {            dp[u][i] = temp[i];        }    }}

总结
问题类型 核心约束 最优解法 时间复杂度
0/1 背包 物品选 0 或 1 次 一维滚动数组 O(n*M)
完全背包 物品无限选 一维正序遍历 O(n*M)
多重背包 物品最多选 k 次 二进制优化 / 单调队列 O(nMlog k) 或 O(n*M)
树形背包 树结构连通块 子树合并 DP O(n²)

每种优化均基于对状态转移的深入分析，通过调整遍历顺序、拆分问题或利用数据结构（如单调队列）降低时间 / 空间复杂度。

背包问题优化详解

（二进制优化与单调队列优化）

一、多重背包的二进制优化

1. 为什么需要二进制优化？

  • 暴力法的缺陷：当物品数量k很大（如k=1e5）时，暴力拆分（拆成k个 0/1 物品）会导致时间复杂度飙升（O(nMk)），无法实用。

  • 核心思想：用 “二进制拆分” 将k个物品转化为log2(k)个 “虚拟物品”，通过组合这些虚拟物品，覆盖所有可能的取用数量（1~k），大幅减少计算量。

2. 二进制拆分的数学原理

  • 整数的二进制表示：任何正整数k都可以唯一分解为若干个 “2 的幂次” 之和，即：

k = 2^0 + 2^1 + 2^2 + ... + 2^t + r

其中r是剩余值（0 ≤ r < 2^(t+1)），t是满足2^0 + ... + 2^t ≤ k的最大整数。

  • 举例：

  • k=10：10 = 1(2^0) + 2(2^1) + 4(2^2) + 3(r)（1+2+4=7 ≤10，剩余3）。

  • 这些拆分的数字（1,2,4,3）可组合出 1~10 的所有整数：

5=1+4，6=2+4，7=1+2+4，8=1+2+5（但 5 不存在，实际8=3+4+1），9=2+3+4，10=1+2+4+3。

3. 二进制优化的步骤

步骤 1：拆分物品数量k

对每个物品(v, w, k)，按二进制拆分k为c_1, c_2, ..., c_m，其中：

  • c_1=1, c_2=2, c_3=4, ...（2 的幂次），直到累加和接近k。

  • 最后一个c_m为剩余值（k - (c_1+...+c_{m-1})）。

步骤 2：生成虚拟物品

每个拆分的c_i对应一个虚拟物品：

  • 体积 = c_i * v

  • 价值 = c_i * w

步骤 3：用 0/1 背包求解

虚拟物品只能选一次，但通过组合可实现 “选 1~k 个原物品” 的效果，用 0/1 背包的一维优化解法即可。

4. 代码推演示例

假设物品为(v=2, w=5, k=10)，背包容量M=20：

拆分k=10

  • 拆分结果：1,2,4,3（1+2+4=7 ≤10，剩余3）。

  • 虚拟物品：

(21=2, 51=5)、(4,10)、(8,20)、(6,15)。

处理虚拟物品（0/1 背包）

int dp[21] = {0}; // 容量0~20// 处理虚拟物品(2,5)for (int m=20; m>=2; m--) dp[m] = max(dp[m], dp[m-2]+5);// 处理虚拟物品(4,10)for (int m=20; m>=4; m--) dp[m] = max(dp[m], dp[m-4]+10);// 处理虚拟物品(8,20)for (int m=20; m>=8; m--) dp[m] = max(dp[m], dp[m-8]+20);// 处理虚拟物品(6,15)for (int m=20; m>=6; m--) dp[m] = max(dp[m], dp[m-6]+15);
  • 结果：dp[20]对应 “选 10 个原物品” 的价值（10*5=50），通过虚拟物品组合（4+6=10体积，20+15=35？不，实际8+6+4+2=20体积对应20+15+10+5=50）。

5. 时间复杂度分析

  • 每个物品拆分后有log2(k)个虚拟物品（如k=1e5需约 17 个）。

  • 总时间复杂度：O(nMlog k)，远优于暴力法的O(nMk)。

二、多重背包的单调队列优化

1. 为什么需要单调队列优化？

  • 二进制优化的局限：当n和M极大（如n=1e3, M=1e4），O(nMlog k)仍可能超时。

  • 核心目标：将时间复杂度降至O(n*M)，完全摆脱对k的依赖。

2. 单调队列优化的数学原理

状态转移的重构

对物品(v, w, k)，容量m的转移方程为：

dp[m] = max{ dp[m - tv] + tw | 0 ≤ t ≤ min(k, m//v) }

按余数分组

将容量m按 “m mod v” 分组，即m = r + t*v（r为余数，0 ≤ r < v；t为倍数，t ≥0）。

  • 例如v=3，r=0时，m=0,3,6,9,...（t=0,1,2,3,...）。

转移方程变形

对组内容量m = r + t*v，转移方程可改写为：

dp[r + t*v] = max{ dp[r + p*v] + (t - p)*w | p ≤ t 且 t-p ≤k }= max{ (dp[r + p*v] - p*w) } + t*w  （其中`p`的范围是 `t -k ≤ p ≤ t`，即窗口大小为`k+1`）
  • 令val(p) = dp[r + pv] - pw，则问题转化为：对每个t，求窗口[t-k, t]内val(p)的最大值，再加上t*w。

3. 单调队列的作用

  • 单调队列：一种维护队列内元素单调递减（或递增）的数据结构，可在O(1)时间内获取区间最大值。

  • 操作：

  1. 新元素入队时，移除队列中所有小于它的元素（保证单调性）。

  2. 移除超出窗口范围（p < t -k）的元素。

  3. 队首元素即为当前窗口的最大值。

4. 单调队列优化的步骤

步骤 1：按余数分组遍历

对物品(v, w, k)，枚举所有余数r（0 ≤ r < v）。

步骤 2：组内按倍数t遍历

对每个r，遍历t（t=0,1,...），计算容量m = r + t*v（m ≤ M）。

步骤 3：维护单调队列

  • 计算val(t) = dp_prev[r + tv] - tw（dp_prev为上一轮结果）。

  • 将val(t)入队，维护队列单调性。

  • 移除窗口外的元素（p < t -k）。

步骤 4：更新当前dp值

dp[r + tv] = max(dp[r + tv], 队首val(p) + t*w)。

5. 代码推演示例

假设物品为(v=3, w=5, k=2)，背包容量M=10，初始dp_prev为上一轮结果。

按余数r=0分组（m=0,3,6,9）

  • t=0：m=0，val(0) = dp_prev[0] - 05 = 0，队列[0]，dp[0] = 0 + 05 = 0。

  • t=1：m=3，val(1) = dp_prev[3] -1*5（假设dp_prev[3]=0，则val= -5）。

入队后队列[0, -5]（窗口[1-2= -1,1]），队首0，dp[3] = 0 +1*5=5。

  • t=2：m=6，val(2)=dp_prev[6]-2*5（假设dp_prev[6]=0，val=-10）。

入队后队列[0, -5, -10]（窗口[0,2]），队首0，dp[6] =0 +2*5=10。

  • t=3：m=9，val(3)=dp_prev[9]-3*5（假设dp_prev[9]=0，val=-15）。

窗口[1,3]，移除p=0（0 < 3-2=1），队列[-5, -10, -15]，队首-5，dp[9] =-5 +3*5=10（对应t-p=3-1=2个物品，符合k=2）。

6. 核心代码

int dp[10005], prev_dp[10005];for (int i = 1; i <= n; i++) {    int v = v[i], w = w[i], k = k[i];    memcpy(prev_dp, dp, sizeof(dp)); // 保存上一轮结果    for (int r = 0; r < v; r++) { // 按余数分组        deque<int> q; // 存储倍数p，维护val(p)的最大值        for (int t = 0; ; t++) { // 倍数t            int m = r + t * v; // 当前容量            if (m > M) break;            // 计算当前val(t)            int val = prev_dp[m] - t * w;            // 维护队列单调性（移除小于val的元素）            while (!q.empty() && (prev_dp[r + q.front() * v] - q.front() * w) <= val) {                q.pop_front();            }            q.push_front(t);            // 移除窗口外的元素（p < t -k）            while (q.back() < t - k) {                q.pop_back();            }            // 更新dp[m]            int best_p = q.back();            dp[m] = max(dp[m], prev_dp[r + best_p * v] + (t - best_p) * w);        }    }}

7. 时间复杂度分析

  • 每个物品需遍历v个余数组，每组遍历M/v个容量，总操作次数为O(nM)（v(M/v) = M，累加后为n*M）。

  • 适用于n和M较大（如1e4级别）且k极大的场景。

总结：两种优化的对比
优化方法 核心思想 时间复杂度 实现难度 适用场景
二进制优化 拆分k为log k个虚拟物品 O(nMlog k) 简单 k中等（如10 < k ≤ 1e4）
单调队列 分组 + 滑动窗口最大值 O(n*M) 较难 k极大或M极大（如1e4+）

  • 实际应用中，二进制优化因实现简单、效率均衡，是多重背包的首选解法。

  • 单调队列优化更适合大规模问题，需深入理解 “余数分组” 和 “滑动窗口” 的逻辑。

状态压缩 DP 状态可用二进制表示，集合大小较小（通常 n≤20），需枚举子集或状态位 旅行商问题（TSP）、拓扑序计数、集合划分问题、蒙德里安的梦想（铺砖问题）
树形 DP 问题基于树结构，需从子树向父节点转移状态，通常采用后序遍历 树的直径、树上最大独立集、二叉树着色问题、树上背包
数位 DP 问题与数字的位数相关，需统计满足特定条件的数字数量，常涉及数字范围限制 计数 1~N 中不含连续相同数字的数、数字中含特定数字的个数、数字和为 K 的数的数量
概率 DP 问题涉及随机事件，需计算概率或期望，状态转移包含概率权重 期望次数问题（如掷骰子到达目标的期望步数）、游戏胜率计算、概率型路径规划
记忆化搜索（递归 DP） 问题可分解为重叠子问题，但递归结构更自然，适合状态复杂或难以线性枚举的场景 滑雪问题、网格路径计数（含障碍）、复杂条件下的子序列问题
二维 DP 状态需用两个维度描述（如 i 和 j），子问题依赖于二维参数的组合 最长公共子序列（LCS）、编辑距离、不同路径（网格 DP）、二维费用背包
滚动数组优化 DP 状态转移仅依赖最近的几个阶段，可压缩空间复杂度（如从 O (n²) 降至 O (n)） 斐波那契数列（空间 O (1)）、0-1 背包（空间 O (m)）、二维 DP 的空间优化
插头 DP 问题涉及网格中的连通性或路径闭合，需用 "插头" 表示边界状态，适合复杂网格划分 网格中有效回路计数、多米诺骨牌覆盖计数、管道铺设问题

这些 DP 类型的核心思想均为将复杂问题分解为重叠子问题，通过存储子问题的解避免重复计算
