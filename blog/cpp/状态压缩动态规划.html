状态压缩动态规划

位运算

• 表示集合的方法有两种：第一种是记录取了哪些元素，第二种是记录每个元素 取或不取。这里采用第二种方法。 • 我们使用一个十进制数s来表示一个集合。 • 将s划为二进制位后，对应数位上为0，则表示这个数不取。为1，则表示取。

• 如，{1,3,4}，可以用1101表示，即13 • {2,5,8,9}可用110010010表示，即402 • 为了表示1..20的数的所有可能集合，需要用到的数是0..(2^20)-1

一个int由32位组成，两个整数的位运算，就是将这两个整数逐位运算

• 1100 & 1010 = 1000 • 1100 | 1010 = 1110 • 1100 ^ 1010 = 0110

左移/右移运算：

• <<：左移运算。a << b表示将a的二进制从右边添b个0。 • 如101 << 3 = 101000 • >>：右移运算。a >> b表示将a的二进制最右面的b位去掉。 • 如10101 >> 3 = 1

查询、赋值

if ((s & (1 << k)) == 0) s的右起第k位是0
else s的右起第k位是1
把s的右起第k位赋为1：
s = s | (1 << k);

状态压缩动态规划

  • 状态压缩dp就是把状态压成二进制位的dp.

  • 一个数s对应二进制位是0/1分别表示不同的状态.

e.g.：拓扑序个数问题

给你一张拓扑图,求这张拓扑图有多少种不同的拓扑序. n<=20

dp[s]表示当前s集合中的点都已经在拓扑序中的方案数,转移考虑枚举下一个点 选什么,下一个选的点要满足它在s中的点选完后的入度为0,转移到dp[s|(1<<i-1)]

动态规划思路解析

  1. 状态定义：

    • dp[s]表示当前状态为s时的拓扑序方案数
    • 其中s是一个二进制数，表示哪些顶点已经被加入到拓扑序中
    • 例如，s = 0b101表示第 1 个和第 3 个顶点已经被选中（假设顶点从 1 开始编号）

  2. 状态转移：

    • 对于每个状态s，我们需要考虑下一个可以加入拓扑序的顶点i
    • 顶点i必须满足：它所有的前驱顶点都已经包含在s中（即此时顶点i的入度为 0）
    • 当选择顶点i后，新的状态为s | (1 << (i-1))，表示将顶点i加入到已选集合中
    • 转移方程：dp[s | (1 << (i-1))] += dp[s]

  3. 初始状态：

    • dp[0] = 1，表示空集合有一种方案（什么都不选）

  4. 最终答案：

    • dp[(1 << n) - 1]，表示所有顶点都被选中时的方案总数

实现要点

  • 需要预处理每个顶点的前驱集合，方便快速判断某个顶点是否可以被选中
  • 遍历所有可能的状态s（从 0 到 2^n-1）
  • 对于每个状态，找出所有可以作为下一个顶点的节点并更新状态

这种方法的时间复杂度是 O (n・2^n)，对于 n≤20 是可行的（2^20 约为 100 万）。

#include <iostream>
#include <vector>
#include <cstring>
using namespace std;

// 计算拓扑序的数量
long long countTopologicalOrders(int n, const vector<vector<int>>& adj) {
    // 预处理每个节点的前驱集合
    vector<int> pre(n + 1, 0);  // 用位表示前驱节点
    for (int u = 1; u <= n; ++u) {
        for (int v : adj[u]) {
            pre[v] |= (1 << (u - 1));  // u是v的前驱
        }
    }

    // dp[s]表示状态s时的拓扑序数量
    vector<long long> dp(1 << n, 0);
    dp[0] = 1;  // 初始状态：空集

    // 遍历所有状态
    for (int s = 0; s < (1 << n); ++s) {
        // 尝试添加每个可能的下一个节点
        for (int i = 1; i <= n; ++i) {
            // 节点i未被选中，且其所有前驱都已被选中
            if (!(s & (1 << (i - 1))) && (pre[i] & s) == pre[i]) {
                int next_s = s | (1 << (i - 1));
                dp[next_s] += dp[s];
            }
        }
    }

    return dp[(1 << n) - 1];  // 所有节点都被选中的状态
}

int main() {
    int n, m;
    cin >> n >> m;

    vector<vector<int>> adj(n + 1);  // 邻接表表示图
    for (int i = 0; i < m; ++i) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);  // u -> v的有向边
    }

    cout << countTopologicalOrders(n, adj) << endl;

    return 0;
}
  • 首先通过邻接表构建图，并预处理每个节点的前驱集合
  • 使用一个整数pre[i]的二进制表示来存储节点i的所有前驱节点
  • 对于每个状态s，检查每个未被选中的节点i是否满足前驱条件
  • 如果满足条件，就更新新状态的方案数
  • 最终返回所有节点都被选中时的方案总数
DP 类型 适用问题特征 典型问题示例
线性 DP 问题具有线性阶段划分，状态按顺序转移，前一状态影响后一状态 最长上升子序列、数字三角形、斐波那契数列、打家劫舍系列问题
区间 DP 问题与区间相关，可通过划分区间为子区间求解，区间长度通常作为阶段 最长回文子序列、矩阵链乘法、石子合并、括号匹配问题
背包 DP 存在 "选择物品" 和 "限制条件"（如重量、体积），需优化目标函数（如价值最大化） 0-1 背包、完全背包、多重背包、分组背包、二维背包
状态压缩 DP 状态可用二进制表示，集合大小较小（通常 n≤20），需枚举子集或状态位 旅行商问题（TSP）、拓扑序计数、集合划分问题、蒙德里安的梦想（铺砖问题）
树形 DP 问题基于树结构，需从子树向父节点转移状态，通常采用后序遍历 树的直径、树上最大独立集、二叉树着色问题、树上背包
数位 DP 问题与数字的位数相关，需统计满足特定条件的数字数量，常涉及数字范围限制 计数 1~N 中不含连续相同数字的数、数字中含特定数字的个数、数字和为 K 的数的数量
概率 DP 问题涉及随机事件，需计算概率或期望，状态转移包含概率权重 期望次数问题（如掷骰子到达目标的期望步数）、游戏胜率计算、概率型路径规划
记忆化搜索（递归 DP） 问题可分解为重叠子问题，但递归结构更自然，适合状态复杂或难以线性枚举的场景 滑雪问题、网格路径计数（含障碍）、复杂条件下的子序列问题
二维 DP 状态需用两个维度描述（如 i 和 j），子问题依赖于二维参数的组合 最长公共子序列（LCS）、编辑距离、不同路径（网格 DP）、二维费用背包
滚动数组优化 DP 状态转移仅依赖最近的几个阶段，可压缩空间复杂度（如从 O (n²) 降至 O (n)） 斐波那契数列（空间 O (1)）、0-1 背包（空间 O (m)）、二维 DP 的空间优化
插头 DP 问题涉及网格中的连通性或路径闭合，需用 "插头" 表示边界状态，适合复杂网格划分 网格中有效回路计数、多米诺骨牌覆盖计数、管道铺设问题

这些 DP 类型的核心思想均为将复杂问题分解为重叠子问题，通过存储子问题的解避免重复计算。
