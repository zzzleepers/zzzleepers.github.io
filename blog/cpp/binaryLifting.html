二进制倍增

问题描述

共有 n 个空间站，编号为 0 … n-1。 每座空间站都配备一条 单向跃迁通道，可以把飞船瞬间传送到另一座空间站。 对每座空间站 i，已知：

f[i] —— 这条通道通向的目标站编号（0 ≤ f[i] ≤ n-1）

w[i] —— 完成这次跃迁所需的能量值（正整数）

保证 每座空间站恰好有且仅有一条跃迁通道（因此一共也是 n 条通道）。

某科研团队准备对这套通道网络进行长距离测试。他们选定一个固定的跃迁次数 k（k ≤ 10^10），并从每一座空间站单独出发，沿着通道连续跃迁恰好 k 次。

对每座起点空间站 i，请你计算两项指标：

总能量消耗 si： 从 i 出发连续跃迁 k 次过程中，所有跃迁能量 w[·] 的总和。

最低单跳能量 mi： 从 i 出发连续跃迁 k 次过程中，所经历的 k 条通道里最小的一次能量消耗。

输入格式

第一行两个数 n 和 k

第二行 n 个数 f(i)

第三行 n 个数 w(i)

输出格式

每行两个数 si 和 mi

输入输出样例

样例1

输入样例
7 3 
1 2 3 4 3 2 6 
6 3 1 4 2 2 3
输出样例
10 1 
8 1 
7 1 
10 2 
8 2 
7 1 
9 3

数据范围与提示

30%的数据：n,k<=1000。

100%的数据：N<=10^5,k<=10^10,0<=f(i)<n,w(i)<=10^8。

二进制倍增法（Binary Lifting）

专门用于高效处理在有向图中需要进行大量重复跳转（如k次跳转）的问题，尤其适合每个节点只有一条出边的特殊结构（如本题中的空间站网络）。

核心思想

将大的跳转次数k分解为2的幂次之和（二进制表示），通过预处理不同2^j步的跳转信息，快速组合出k步的结果。例如，k=13（二进制1101）可以分解为8+4+1，即2^3 + 2^2 + 2^0。

数组定义

  • f[i][j]：表示从节点i出发，连续跳转2^j步后到达的节点

  • w[i][j]：表示从节点i出发，连续跳转2^j步过程中经历的最小能量值（根据题目需求，也可以是最大能量值）

  • 类似地，还可以定义s[i][j]：表示从节点i出发，连续跳转2^j步的总能量消耗

    解决了大规模数据的效率问题。

预处理过程

  1. 基础情况（j=0）：
    • f[i][0] = f[i]（跳转1步，即2^0步）
    • w[i][0] = w[i]（1步的最小能量）
    • s[i][0] = w[i]（1步的总能量）
  2. 递推关系（j>0）：
    • 跳转2^j步 = 先跳转2^(j-1)步，再跳转2^(j-1)步
    • f[i][j] = f[ f[i][j-1] ][j-1]
    • w[i][j] = min( w[i][j-1], w[ f[i][j-1] ][j-1] )（两次跳转中的最小能量）
    • s[i][j] = s[i][j-1] + s[ f[i][j-1] ][j-1]（两次跳转的总能量和）

计算k步结果

对于每个起点i，根据k的二进制表示，将k分解为多个2^j的和，然后依次累加对应的能量和与最小能量：

  • 初始化当前节点为i，总能量sum=0，最小能量min_w=∞

  • 遍历k的二进制每一位，若第j位为1，则：

    • sum += s[current][j]
    • min_w = min(min_w, w[current][j])
    • current = f[current][j]

  • 最终sum和min_w即为k步的结果

优势

  • 预处理时间复杂度：O(n log k)（n为节点数，log k为最大幂次）

  • 单次查询时间复杂度：O(log k)

  • 能高效处理k高达10^10的情况，避免超时 这种方法非常适合本题场景，二进制分解将原本需要O(k)的模拟过程优化为O(log k)

#include <bits/stdc++.h>  // 包含标准库的所有头文件
#define ll long long     // 定义ll为long long的别名，简化代码书写
using namespace std;

const int N=100005;      // 定义常量N，表示最大空间站数量
const int inf=1000000000;// 定义一个较大的整数作为初始最小值

int x,y,mnl,n;           // x,y用于临时存储输入数据，mnl存储最小单跳能量，n是空间站数量
ll k,s;                  // k是跃迁次数，s存储总能量消耗
// 三维数组分别存储：f[j][i]表示从j出发跳2^i次到达的空间站
// mn[j][i]表示从j出发跳2^i次过程中的最小能量
// g[j][i]表示从j出发跳2^i次的总能量
int mn[N][35], f[N][35];
ll g[N][35];

// 初始化函数，预处理跳2^i次的相关信息
void init(){
    // 从1到33（因为2^33已经超过10^10）
    for (int i=1;i<=33;i++)
        // 对每个空间站j进行处理
        for (int j=1;j<=n;j++){
            // 跳2^i次 = 先跳2^(i-1)次，再从到达的位置跳2^(i-1)次
            f[j][i] = f[f[j][i-1]][i-1];
            // 2^i次跳跃中的最小能量 = 前2^(i-1)次和后2^(i-1)次中的最小值
            mn[j][i] = min(mn[j][i-1], mn[f[j][i-1]][i-1]);
            // 2^i次跳跃的总能量 = 前2^(i-1)次和后2^(i-1)次的能量之和
            g[j][i] = g[j][i-1] + g[f[j][i-1]][i-1];
        }
}

int main(){
    // 读取空间站数量n和跃迁次数k
    scanf("%d%lld",&n,&k);

    // 读取每个空间站的跃迁目标
    for (int i=1;i<=n;i++){
        scanf("%d",&x);  // 读取目标空间站编号
        x++;             // 将编号从0-based转换为1-based（便于数组索引）
        f[i][0] = x;     // 存储跳2^0=1次到达的空间站
    }

    // 读取每个空间站的跃迁能量
    for (int i=1;i<=n;i++){
        scanf("%d",&y);          // 读取能量值
        mn[i][0] = y;            // 存储跳1次的最小能量（就是当前跳跃的能量）
        g[i][0] = 1ll * y;       // 存储跳1次的总能量（转换为long long类型）
    }

    // 调用初始化函数，预处理所有2^i次跳跃的信息
    init();

    // 对每个空间站作为起点进行计算
    for (int i=1;i<=n;i++){
        s = 0;              // 初始化总能量为0
        mnl = inf;          // 初始化最小单跳能量为一个很大的值
        int x = i;          // 当前所在的空间站

        // 从最高位到最低位检查k的二进制表示
        for (int j=33;j>=0;j--){
            // 如果k的第j位为1，则需要跳2^j次
            if ((k >> 1ll * j) & 1){
                s += g[x][j];             // 累加2^j次跳跃的总能量
                mnl = min(mnl, mn[x][j]); // 更新最小单跳能量
                x = f[x][j];              // 更新当前所在的空间站位置
            }
        }

        // 输出总能量和最小单跳能量
        printf("%lld %d\n", s, mnl);
    }

    return 0;
}

↓这段代码是整个程序的核心计算部分，负责根据预处理的倍增数据计算出每个起点空间站经过k次跃迁后的总能量消耗和最小单跳能量。具体解释如下：

 // 如果k的第j位为1，则需要跳2^j次
 if ((k >> 1ll * j) & 1){

 }

这段代码的工作原理基于二进制分解思想：

  1. 任何整数k都可以表示为2的幂次之和（二进制表示）
  2. 例如k=5（二进制101）=4+1=2²+2⁰ 循环从最高位（33位）向最低位（0位）检查：
  • 当k的第j位为1时，意味着需要进行2^j次跃迁
  • 此时使用预处理好的数组：
    • g[x][j]提供从当前位置x跳2^j次的总能量
    • mn[x][j]提供这2^j次跳跃中的最小单跳能量
    • f[x][j]记录跳2^j次后到达的位置

通过这种方式，原本需要O(k)时间的计算被优化为O(log k)，高效处理了k高达10^10的情况。

输入的不是10进制吗，从哪里开始突然变成二进制了？

代码中并没有将输入的数值从十进制转换为二进制，而是利用了二进制的数学性质来优化计算。这是一种算法思想，而非数据格式的转换。 具体来说，代码始终使用十进制数值进行计算，但借鉴了二进制的表示特点： 任何整数都可以分解为2的幂次之和，例如：

  • 5（十进制）= 4 + 1 = 2² + 2⁰（二进制表示为101）

  • 7（十进制）= 4 + 2 + 1 = 2² + 2¹ + 2⁰（二进制表示为111） 在这段代码中：

    for (int j=33;j>=0;j--)
if ((k >> 1ll * j) & 1) { ... }

    (k >> j) & 1 是一种位运算，用于检查十进制数k的二进制表示中第j位是否为1。这只是一种计算技巧，用来高效分解k的组成，而不是将k真正转换为二进制格式。 这种方法的优势在于：

  • 把k次跳跃分解为若干个2^j次跳跃的组合

  • 原本需要O(k)的时间复杂度被优化为O(log k)

  • 特别适合处理k很大（如10^10）的情况 简单说，这是用二进制的思想解决十进制的问题，是算法设计中的常见优化手段。
